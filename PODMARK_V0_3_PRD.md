# PodMark v0.3 PRD

## 1. 产品定位

PodMark 是一个面向播客重度用户的播客文章阅读与学习工具。

用户输入一集播客链接后，PodMark 将播客整理成一篇可以阅读、标注、回听和收藏的播客文章。用户可以在阅读过程中划线收藏、添加灵感标签、写阅读笔记，并把这些学习痕迹沉淀到自己的学习库中。

PodMark 的核心不是“播客转写工具”，也不是“AI 总结工具”，而是：

> 把值得记住的播客，变成可以反复阅读和沉淀的学习对象。

当前首页 slogan：

```text
Mark the podcasts worth remembering.
```

## 2. 目标用户

第一版优先面向以下用户：

- 经常听长播客，但听完后很难复盘的人
- 喜欢从播客中获取思想、案例、观点和灵感的知识工作者
- 希望把音频内容转化为个人知识库的学习型用户
- 需要作品集级产品 Demo 的评审者或潜在用户

第一版不追求覆盖所有播客用户，而是服务“听播客是为了学习和沉淀”的人。

## 3. 核心概念

### 播客文章

播客文章是 PodMark 的核心内容对象。

它不是原始字幕，也不是普通文字稿，而是经过整理、适合阅读、可以标注和收藏的播客内容文章。

### 正在阅读

用户当前打开并深度阅读的一篇播客文章。

该页面承担主要学习行为，包括阅读、回听、划线收藏、标签和笔记。

### 学习库

学习库是用户所有播客文章和学习痕迹的沉淀空间。

它不是简单列表，而是一个带有最近学习、全部收藏、进度和学习痕迹统计的知识库首页。

### AI 学习助手

AI 学习助手用于辅助用户理解播客文章，包括摘要、大纲和金句。

AI 是辅助入口，不是主角。它帮助用户进入阅读，而不是替代用户阅读。

### 学习痕迹

学习痕迹包括：

- 划线收藏
- 灵感标签
- 阅读笔记
- 学习进度

学习痕迹需要能从学习库回到原文上下文。

## 4. 信息架构

第一版保留三个核心页面：

```text
/
首页

/podcasts/[id]
正在阅读页

/library
学习库
```

顶部导航统一为：

```text
首页 / 正在阅读 / 学习库
```

导航在不同页面中保持固定位置和固定顺序，避免切换页面时跳动。

## 5. 首页

### 页面目标

首页需要让用户快速理解：

```text
PodMark 可以把播客变成值得保存、阅读和标注的播客文章。
```

### 页面模块

- 顶部导航
- 品牌 Logo
- 英文 slogan
- 中文产品说明
- 播客链接输入卡片
- 生成播客文章按钮
- 示例演示入口
- 三个价值卡片：转录、标注、收藏

### 首页主文案

```text
Mark the podcasts worth remembering.

把一集播客整理成可以阅读、标注和收藏的播客文章。
```

### 首页交互

#### 输入播客链接

用户在输入框中粘贴播客链接。

MVP 阶段不接真实解析，点击生成后进入示例播客文章。

```text
输入链接
→ 点击生成播客文章
→ 进入 /podcasts/[id]
```

#### 示例演示

用户点击示例演示后，直接进入示例正在阅读页。

```text
点击示例演示
→ 进入 /podcasts/demo-product-thinking
```

### 首页当前边界

第一版首页不展示复杂生成流程，不展示真实解析状态，不展示转写队列。

## 6. 正在阅读页

### 页面目标

正在阅读页是 PodMark 的核心页面。

它的目标是让用户像阅读文章一样研读播客，并在重要段落上形成学习痕迹。

### 页面结构

#### 左侧：播客文章阅读区

包含：

- 播客封面
- 分类 / 节目集数 / 时长
- 播客文章标题
- 引言
- 分段正文
- 段落时间点
- 当前阅读段落
- 已划线高亮

正文应该是页面主角。时间点只是辅助定位，不作为产品亮点。

#### 右侧：辅助与学习区

包含：

- AI 学习助手
  - 摘要
  - 大纲
  - 金句

- 我的学习
  - 划线收藏
  - 灵感标签
  - 阅读笔记

#### 底部：悬浮播放器

播放器用于支持回听，不应压过阅读。

包含：

- 当前播放信息
- 后退
- 播放 / 暂停
- 快进
- 播放进度
- 倍速
- 音量

### 正在阅读页交互

#### 点击时间点

```text
点击段落时间点
→ 播放器跳转到对应时间
→ 当前段落进入 active 状态
```

MVP 可先实现滚动定位，再实现音频联动。

#### 选中文字

```text
选中文章文字
→ 出现标注工具
→ 用户可选择划线收藏 / 添加标签 / 写阅读笔记
```

第一版建议限制用户只能在单个段落内选择文字。

#### 划线收藏

```text
选中文字
→ 点击划线收藏
→ 原文出现高亮
→ 右侧我的学习同步出现
→ 学习库中统计数量更新
```

#### 灵感标签

```text
选中文字或段落
→ 添加标签
→ 标签归档到当前播客文章
→ 学习库可按标签筛选
```

#### 阅读笔记

```text
选中文字
→ 点击写笔记
→ 出现轻量笔记编辑器
→ 显示原文摘录
→ 用户输入自己的理解
→ 自动保存到我的学习
```

#### 右侧学习痕迹

```text
点击右侧划线收藏 / 笔记
→ 左侧正文滚动到对应段落
→ 对应段落短暂高亮
```

#### AI 学习助手

```text
点击 摘要 / 大纲 / 金句
→ 切换对应内容
```

后续大纲和金句应支持点击跳转到正文对应段落。

## 7. 学习库

### 页面目标

学习库用于沉淀用户的播客文章和学习痕迹。

它不是简单的播客列表，而是用户的个人播客知识库。

### 页面结构

#### 顶部

- 统一导航
- 搜索框
- 账户入口

#### 学习库 Hero

包含：

- 页面标题
- 简短说明

#### 最近学习

展示最近打开或学习过的播客文章。

卡片包含：

- 封面
- 分类
- 标题
- 摘录
- 学习进度
- 划线收藏数
- 灵感标签数
- 阅读笔记数

#### 全部收藏

展示所有收藏的播客文章或学习内容。

支持筛选：

- 全部收藏
- 进行中
- 已完成
- 哲学
- 技术
- 更多

### 学习库交互

#### 点击最近学习卡片

```text
点击卡片
→ 进入对应正在阅读页
```

#### 点击全部收藏卡片

```text
点击卡片
→ 进入对应正在阅读页
```

#### 搜索

```text
输入关键词
→ 搜索播客文章标题、摘录、笔记、标签
```

MVP 可先做前端 mock 搜索。

#### 筛选

```text
点击筛选项
→ 切换列表内容
```

MVP 可先做 UI 状态，后续再接真实数据。

#### 加载更多

```text
点击加载更多探索
→ 加载更多收藏内容
```

MVP 可先追加 mock 数据。

## 8. 数据模型建议

### PodcastArticle

```ts
type PodcastArticle = {
  id: string
  sourceUrl: string
  title: string
  showName: string
  episode?: string
  coverUrl?: string
  duration?: number
  category?: string
  quote?: string
  articleBlocks: ArticleBlock[]
  aiAssistant: AIAssistant
  progress: number
  createdAt: string
  updatedAt: string
}
```

### ArticleBlock

```ts
type ArticleBlock = {
  id: string
  time?: number
  text: string
}
```

### AIAssistant

```ts
type AIAssistant = {
  summary: string[]
  outline: {
    id: string
    title: string
    time?: number
  }[]
  quotes: {
    id: string
    text: string
    time?: number
  }[]
}
```

### LearningMark

```ts
type LearningMark = {
  id: string
  articleId: string
  blockId: string
  selectedText: string
  time?: number
  type: "highlight" | "tag" | "note"
  note?: string
  tags?: string[]
  createdAt: string
  updatedAt: string
}
```

## 9. MVP 范围

### 第一阶段必须完成

- 首页静态体验
- 输入框和示例演示跳转
- 正在阅读页静态内容
- AI 学习助手静态 tab
- 我的学习静态列表
- 学习库静态卡片
- 统一视觉系统
- 页面间导航稳定

### 第二阶段建议完成

- 正在阅读页选中文本工具条
- 单段内划线收藏
- 我的学习同步更新
- 学习库统计同步更新
- localStorage 持久化
- 点击学习痕迹回到正文

### 第三阶段建议完成

- 悬浮播放器真实交互
- 时间点跳转
- 播放进度同步
- 倍速控制

### 第四阶段建议完成

- mock 生成流程
- 播客链接基本校验
- 失败时 fallback 到示例文章

### 第五阶段再探索

- 小宇宙真实解析
- 音频源获取
- 真实转写
- AI 摘要 / 大纲 / 金句生成

## 10. 第一版明确不做

- 不做登录系统
- 不做真实小宇宙解析
- 不做真实 AI 生成
- 不做真实转写
- 不做复杂搜索系统
- 不做社区分享
- 不做多人协作
- 不做移动 App
- 不做浏览器插件
- 不做复杂知识图谱
